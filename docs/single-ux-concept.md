# Single UX Concept

## Product sentence

`Days verwandelt Gedanken, Signale und Ziele in einen klaren Tag.`

Use that sentence as the test for every visible `Single` change.

## Desired feeling

`Single` should feel:
- calm
- local
- private
- fast to read
- direct to correct

`Single` should not feel like:
- a collection of mini apps
- a tool hub
- a self-rating dashboard
- a settings-heavy lab

## Home is the anchor

`Home` remains the visual and functional center.

Rules:
- `Home` stays recognizable.
- The day track is the first read.
- The day is shown in three broad windows first.
- The surface stays text-light and visually calm.

## Core movement

1. Open `Home`.
2. Read fit or drift.
3. Tap a block.
4. Inspect just enough detail.
5. Make one correction.
6. Return to `Home` and see the day change.

## Interaction rules

### Day track

The graph should answer:
- how full the day wants to be
- how full it currently is
- where it drifts

### Block overlay

The overlay is the main action layer.

It shows:
- block focus
- `Soll`
- `Ist`
- reasons
- current items
- direct actions

### Quick add

`+` stays secondary.

It supports:
- quick capture
- calibration
- setup and inspection entry points

It must not become the main value carrier.

## Navigation rules

- `Home` remains the stable anchor.
- Back closes overlays before screen navigation.
- Share/import returns to `Home`.
- Optional calibration such as `Lebensrad` must never block first use.

## Near-term constraints

For the next iterations:
- keep the current Home geometry
- improve meaning before adding more tools
- keep the visible surface minimal
- prefer one strong loop over breadth
- let passive signals and local learning feed Home later, not clutter it now
